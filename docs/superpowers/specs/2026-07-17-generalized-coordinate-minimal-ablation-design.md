# 广义坐标误差最小因果消融设计

日期：2026-07-17

## 1. 目标与研究问题

本实验优先验证最初的广义坐标误差假说：在保持因果 Mamba 主干、训练预算和任务不变的前提下，把时间对齐的零阶、一阶和二阶预测误差用于调制 selective SSM，是否能够稳定改善连续动力学任务，尤其是长度、动力学参数和噪声分布外泛化。

本实验不用于证明完整的迭代 Predictive Coding Network。它检验的范围严格限定为：当前共享权重双遍、误差条件化 Mamba 中，增加广义坐标误差阶数是否带来可归因的收益。

若假说未通过预先规定的门槛，保留完整负结果和实验分支，并回到当前 query-conditioned Temporal Mamba 基线 checkpoint，继续组合时序逻辑方向。不得在看到结果后修改通过门槛。

## 2. 范围与不做事项

### 2.1 本阶段包含

- 因果、时间对齐的 K=1、K=2、K=3 预测误差。
- 参数量匹配的无误差、误差阶数、误差打乱和等方差噪声消融。
- 可控二阶动力系统和 UCI HAR 连续传感器两个数据域。
- ID、长度 OOD、动力学参数 OOD 和噪声 OOD 评估。
- 三种子初筛与五种子确认。
- Git、数据 manifest、运行环境和结果文件的严格溯源。

### 2.2 本阶段不包含

- query binder 或时序逻辑数据集训练。
- Predictor 前置于 Mamba 的替代架构。
- 多层迭代 PCN、自由能目标或多轮状态校正。
- 对 Mamba 的 B、C 或 A 矩阵增加新的误差注入路径。
- 注意力、双向编码、非因果差分或新的读出结构。
- 为追求正结果而进行未预注册的架构搜索。

## 3. 模型架构

### 3.1 固定主干

固定当前 `temporal_mamba` 的直接因果 selective recurrence、共享权重双遍编码、分类头、优化器、数值保护和有界 `dt` 调制。第一遍编码不接收误差，第二遍使用相同的 Mamba 参数。

局部代码变更必须隔离为广义坐标 Predictor、误差构造器、阶数 mask 和独立实验配置。现有 temporal-logic v1/v2 和 UCI HAR 默认配置的行为不得改变。

### 3.2 K=3 Predictor

第一遍隐藏状态 `H1[:, t]` 通过三个独立预测头，预测下一时刻的：

- 零阶信号 `x`；
- 一阶因果差分 `x'`；
- 二阶因果差分 `x''`。

所有变体均实例化完整 K=3 Predictor 和相同误差投影。K=1、K=2、K=3 仅通过固定 mask 决定启用哪些阶数，因此模型参数量完全一致。未启用的预测头不进入误差路径或辅助损失。

### 3.3 时间对齐与梯度边界

由时刻 `t` 的第一遍隐藏状态预测时刻 `t+1` 的目标：

- `e0[t+1] = x[t+1] - x_hat[t]`；
- `e1[t+1] = dx[t+1] - dx_hat[t]`；
- `e2[t+1] = ddx[t+1] - ddx_hat[t]`。

序列开头缺少历史的阶数位置使用零 mask，不计入辅助损失，也不得向第二遍注入伪误差。第一遍隐藏状态和预测目标在 Predictor 边界 detach；Predictor 可通过第二遍任务损失和预注册的辅助预测损失学习，但不能通过该路径反向改变第一遍激活图。

### 3.4 导数与归一化

合成数据优先使用生成器提供的解析一阶、二阶状态；同时保存因果有限差分版本用于一致性诊断。UCI HAR 只能使用当前及过去样本的后向差分，不允许中心差分或未来泄漏。

零阶、一阶、二阶目标分别使用训练集统计量标准化。每一阶误差先独立 RMS 归一化，再拼接和投影，避免二阶误差的自然尺度主导 `dt` 调制。

### 3.5 误差注入

误差仅通过当前有界通道调制第二遍 selective SSM 的 `dt`：

- 调制强度从零初始化；
- `dt` 始终限制在现有 `[dt_min, dt_max]`；
- K=1、K=2、K=3 使用同一个投影输出维度和同一个调制上限；
- 不修改 B、C、A、残差、卷积或门控路径。

## 4. 数据设计

### 4.1 可控二阶动力系统

生成三个连续动力学族：阻尼振子、受迫振子和状态切换线性系统。主任务是从观测序列分类动力学类别。生成器必须随机化初始状态、相位、幅值、频率、阻尼和外部驱动，并在类别间尽可能匹配单时刻幅值、全局均值和方差，减少静态统计捷径。

数据划分包括：

- ID：训练参数范围、未见轨迹；
- 长度 OOD：训练长度 128，测试长度 256 和 512；
- 参数 OOD：阻尼、驱动频率或切换速率超出训练区间；
- 噪声 OOD：加入训练未见强度的观测噪声。

固定数据种子生成不可变 manifest。每条样本记录动力学族、生成参数、标签、序列长度和内容指纹。不同划分不得共享轨迹指纹。

### 4.2 UCI HAR

复用仓库现有官方 train/test 划分和数据加载路径，不重新划分测试集。训练统计量只从训练划分计算。除标准测试指标外，增加确定性的长度裁剪/前缀视图和传感器噪声视图；所有变体复用相同视图和样本顺序。

## 5. 消融矩阵

所有变体使用相同 K=3 Predictor、误差适配器、Mamba 主干、分类头、训练轮数、batch、优化器和配对随机种子。

| 变体 | 遍数 | 有效误差 | 目的 |
|---|---:|---|---|
| `vanilla` | 1 | 无 | 基础 Mamba |
| `two_pass` | 2 | 无 | 控制第二遍计算量 |
| `gc_k1` | 2 | `e0` | 零阶预测误差 |
| `gc_k2` | 2 | `e0,e1` | 加入一阶误差 |
| `gc_k3` | 2 | `e0,e1,e2` | 完整最小广义坐标误差 |
| `gc_k3_shuffled` | 2 | 跨样本打乱 K=3 误差 | 检查样本语义对齐 |
| `gc_k3_noise` | 2 | 同批次逐阶匹配均值、方差的高斯噪声 | 控制额外随机调制和尺度 |

误差打乱必须使用确定性随机种子，并保持同一批次内误差张量的边际统计。噪声控制不得读取标签。

## 6. 损失与训练

任务损失沿用各数据集现有分类损失。`gc_k1/gc_k2/gc_k3` 及两种 K=3 控制共享同一个预注册辅助损失权重和 warmup；辅助损失只覆盖该变体启用的有效阶数，并忽略开头无合法导数的位置。

每次运行必须记录：总损失、任务损失、各阶辅助损失、各阶误差 RMS、`dt_min/max`、调制强度、梯度范数、分类指标、运行时间和峰值显存。任何非有限值、`dt` 越界或元数据不匹配均使运行失败，不得纳入汇总。

## 8. 阶段门控

### Stage 0：基线 checkpoint

在修改模型前记录当前 query-conditioned v2 的 Git 提交、数据 manifest、12/12实验报告和远端artifact位置。基线 tag 固定命名为 `checkpoint/query-mamba-v2-before-gc-v1`；广义坐标工作使用 `exp/generalized-coordinate-minimal-ablation` 分支。不得覆盖v2数据和artifact。

### Stage 1：确定性与 smoke

在 Pro 6000 上完成：

1. 解析导数与有限差分一致性测试；
2. 一阶、二阶时间对齐和起始 mask 测试；
3. K mask 与参数量一致性测试；
4. `two_pass` 的误差严格为空；
5. shuffled/noise 控制的统计和确定性测试；
6. 所有变体 tiny-batch 过拟合；
7. 数值有限且 `dt` 不越界；
8. 现有 temporal-logic v1/v2 与 UCI HAR 回归测试通过。

任一项失败时停止训练矩阵，保留失败证据并只修复一个明确根因。

### Stage 2：三种子初筛

两个数据域分别运行七个变体和三个配对种子 `42, 123, 777`。一个数据域通过初筛必须同时满足：K3 相对 `two_pass` 的三种子平均配对差值大于0，并且 K3 在至少两个种子上同时优于 `gc_k3_shuffled` 与 `gc_k3_noise`。两个数据域都未通过初筛时停止扩展并进入失败 checkpoint；至少一个数据域通过时进入 Stage 3。

### Stage 3：五种子确认

扩展到五个预注册种子 `42, 123, 777, 2026, 31415`，完成全部 ID 和 OOD 视图。最终报告使用配对差值、均值、样本标准差和双侧 Student-t 95%置信区间；同时报告所有单种子结果，不能只报告最佳种子或最佳epoch。

## 9. 决策判据

### 9.1 支持假说

K3 必须在两个数据域同时满足：

- 合成动力学主 OOD 指标相对 `two_pass` 提升至少 2.0 个百分点；该指标定义为长度512、参数 OOD、噪声 OOD 三个视图 balanced accuracy 的算术平均；
- UCI HAR 主 OOD 指标相对 `two_pass` 提升至少 1.0 个百分点；该指标定义为50%因果前缀与传感器噪声两个视图 macro-F1 的算术平均；
- 上述配对差值的95%置信区间下界大于0；
- K3 优于 `gc_k3_shuffled` 和 `gc_k3_noise`；
- ID 指标相对 `two_pass` 的下降不超过1.0个百分点；
- 所有运行满足数值和溯源要求。

K3 相对 K1/K2 的结果用于判断高阶贡献。如果 K2 最优但 K3 未通过，则只能支持“一阶广义误差有效”，不能支持完整 K=3 假说。

### 9.2 不确定

只有一个数据域满足完整门槛，或均值达到门槛但置信区间包含0，结论记为不确定。不得将不确定写为正结果；后续是否增加种子必须作为新的预注册实验处理。

### 9.3 不支持并回退

两个数据域均未达到门槛，或 K3 未优于 shuffled/noise 控制，则当前实现下的广义坐标误差假说不受支持。保留代码、配置、artifact和负结果报告，给实验分支建立 `checkpoint/gc-v1-terminated` 终止 tag，然后从 Stage 0 基线 checkpoint 继续 query-conditioned Temporal Mamba，不把广义坐标路径设为默认模型。
